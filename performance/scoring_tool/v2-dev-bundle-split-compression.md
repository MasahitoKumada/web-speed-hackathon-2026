## 考察

- バンドル分割 + gzip 圧縮を実施したが、**FCP/LCP/SI は依然 0点**
- v1 (240点) からむしろスコアが下がっている (229点, 207点)
- 原因分析:
  - **941.js (6.6MB, gzip 2.3MB)** がまだ初期ロードに含まれている (splitChunks の vendor chunk)
  - fly.io の低スペック VM では gzip 圧縮の CPU コストが影響している可能性
  - 2回目は「DM詳細ページ → サインインモーダルの表示に失敗」が発生 → lazy ロードの不安定さ
- **941.js の内訳を分析し、初期ロードに不要なモジュールをさらに分離する必要がある**

## 次のステップ

- 941.js に含まれる重量モジュール (kuromoji, web-llm, highlight.js 等) を dynamic import で分離
- 初期ロードを 1MB 以下に抑える
- 静的ファイルの Cache-Control 設定を最適化
